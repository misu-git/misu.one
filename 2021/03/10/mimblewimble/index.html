Mimblewimble

Mimblewimble is a confidential transactions blockchain protocol.

It allows for a considerably simpler blockchain in which all spent transactions can be cut off.

Essentially, an Elliptic Curve Digital Signature Algorithm (ECDSA) can be considered public key and the differential between outputs, inputs and transaction fees for a valid confidential transaction has to be 0.

A prover who builds a confidential transaction can thus sign the transaction with the public key difference between the outputs and inputs.

This allows that new nodes can validate the whole blockchain efficiently without having to download any previous and used transactions.

The blockchain consists only of blockheaders, remaining UTXO outputs and a non-removable kernel of transactions.

The blockchain is composed of blockheaders.

Mimblewimble also allows the aggregation of transactions before the blockchain is committed.

Although there are strong guarantees of confidence, even in Mimblewimble, all commitments which are unique can still be registered for each transaction received and traced, which entries are transformed into which outputs and therefore the transaction graph efficiently reconstructed.

This can be done easily by adding a network node for monitoring all transactions through the network.

This does not, of course, constitute a significant infringement of privacy since although UTXO has been declared anonymous, the entire history of this user does not become declared anonymous (unlike in Bitcoin).

Mimblewimble basics

  1. No addresses available.

  2. No amounts are available.

  3. 2 transactions can be merged into one block, one spending another in one block, and all intermediary details deleted.

How are we doing that?

A encryption sounds great, but what it means is, “I’m proving I knew before I made it public because I can’t reverse the hash to reveal myself.”

In addition, commitments (for example) are made to demonstrate public forecasts even beyond cryptocurrencies, such as John Galt’s public forecast of the development of projects in 2018 on 1 January.

In 2019, John can reveal his list and people can easily check to see if his list (in plain text in English) corresponds to the previous year.
